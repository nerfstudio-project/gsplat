G-SHARP v0.2 → gsplat Integration Plan

Porting the training-time algorithmic components of G-SHARP v0.2 (surgical 4D Gaussian Splatting) into the gsplat library, natively.
Status: Scaffolding landed, awaiting architecture review Branch: vnath_gsharp Base: nv/main Implementation: not started (TDD) Source: holohub/applications/surgical_scene_recon Target: gsplat/

1. Goals and non-goals

Goals

  • Port all training-time algorithmic contributions of G-SHARP v0.2 into gsplat.
  • Keep deformable / 4D machinery behind a clearly experimental gsplat/contrib/ namespace.
  • Ship test-first: every ported unit has positive + negative pytest cases.
  • Provide a runnable end-to-end tutorial on the EndoNeRF sample.

Non-goals

  • No Holoscan / HoloHub imports anywhere.
  • No Depth Anything V2 / MedSAM3 / VGGT code inside gsplat/. The tutorial only references them as optional upstream preprocessing.
  • No compression / rate-distortion work (not present in G-SHARP v0.2).
  • No camera-pose optimization (G-SHARP v0.2 doesn't do it either).

2. Architecture

core (stable) contrib (experimental) examples docs tests
Why this split? Core library stays pure splatting math; 4D / deformable pieces (HexPlane, DeformNet, DynamicStrategy) are still research-grade, so they land under gsplat.contrib.dynamic with an API-may-change banner — same pattern as torchvision.prototype. The EndoNeRF/SCARED loader is domain glue, so it lives next to colmap.py/ncore.py under examples/datasets/.

6. Open risks

Deformation coupling to rasterization()

gsplat.rasterization has no time axis. DynamicStrategy therefore applies the deform-net to (means, quats, opacities) before rasterization(...) is called — same pattern as G-SHARP's rasterize_splats.

CUDA-only paths

HexPlane / DeformNet tests run on CPU for determinism. GPU smoke tests marked CUDA-only, matching tests/test_rasterization.py.

Dataset licensing

EndoNeRF sample and SCARED are not redistributed in-tree. Tutorial documents the sources.

Optional dependencies

KNN scale init has a pure-torch fallback so sklearn remains optional. No new required deps.

7. Review asks

Before implementation starts, please sign off on:

  1. Layout: core-vs-contrib split above.
  2. Contrib namespace: gsplat/contrib/dynamic/ as the home for 4D / deformable research code.
  3. Examples scope: EndoNeRF loader under examples/datasets/ and the recipe at examples/dynamic_trainer.py.
  4. No external pretrained nets in-library: DA2 / MedSAM3 / VGGT stay out of gsplat/; only referenced in the tutorial.
  5. Test matrix: coverage above is the minimum bar per component.
